# Booking-Service

## Описание.
booking-service - единый сервис бронирования переговорок: администраторы создают переговорки и настраивают расписание их доступности (например, по дням недели и времени), система сама формирует слоты для бронирования на основе расписания. Сотрудники просматривают свободные слоты и создают или отменяют брони.

## Установка и сборка.
Далее подразумевается, что компилятор языка Go и/или Docker уже установлены в систему.
Для сборки и запуска проекта необходимо выполнить следующие команды. 

1) Скачать репозиторий в локальную папку.
```
    git clone https://github.com/cofeekru/booking-service.git
```
Есть два варианта запуска:

2.1) Собрать и запустить локально.
```
    go build ./cmd/main.go && ./main
```

2.2) Запустить c помощью docker-compose в фоновом режиме. 
```
    docker-compose up -d --build
```

## Обзор репозитория.

1) __api__ - директория, содержащая .yaml файл с описанием API

2) __cmd__ - директория, содержащая файл main.go, запускающий сервер 

3) __e2e_tests__ - директория, e2e тесты

4) __internal__ - директория, содержащая исходный код

    4.1) __internal/config__ - директория, содержащая все необходимые типы и структуры, необходимые для работы сервиса

    4.2) __internal/database__ - директория, содержащая файлы для работы с базой данных

    4.3) __internal/handlers__ - директория, содержащая все хэндлеры

    4.4) __internal/mocks__ - директория, содержащая все моки, необходимые для unit-тестов

    4.5) __internal/services__ - директория, содержащая всю бизнес-логику; __internal/services/services_test__ - директория, содержащая unit-тесты для бизнес логики

## Тестирование.

Для e2e-тестов запустить отдельное тестовое окружение:

1) Перейти в e2e_tests директорию
```
    cd e2e_tests
```

2) Запустить отдельное тестовое окружение
```
    docker-compose -f docker-compose.e2e.yaml up -d --build
```
3) Запустить отдельно каждый e2e тест
```
    go test ./create_and_booking/...
```
```
    go test ./cancel_booking/...
```
